# Contributing

از اینکه می‌خوای به این پروژه کمک کنی، ممنونم! 🙌  
این فایل راهنمای مشارکت، تست و ارسال Pull Request است.

> اگر چیزی را نمی‌فهمی یا مشکلی داری، قبل از کدنویسی یک Issue باز کن تا مسیر درست را پیشنهاد بدیم.



## 1) قبل از شروع

### کد بیس را تازه نگه دار
```bash
git checkout main
git pull
```

### یک Branch جدید بساز
```bash
git checkout -b feature/<name>




## 2) راه‌اندازی محیط توسعه

### ایجاد و فعال‌سازی محیط مجازی
```bash
python -m venv .venv
```

- Linux/macOS:
```bash
source .venv/bin/activate
```
- Windows (PowerShell):
```powershell
.venv\Scripts\Activate.ps1
```

### نصب وابستگی‌ها
```bash
pip install -r requirements.txt
```

---

## 3) اجرای پروژه به صورت لوکال

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

> اگر فایل entrypoint متفاوت است (مثلاً `app.py` یا `src/main.py`) قسمت `main:app` را مطابق پروژه‌ات تغییر بده.

---

## 4) تست‌ها

برای اجرای تست‌ها:
```bash
pytest -q
```

> قبل از ارسال PR حتماً تست‌ها را اجرا کن.



## 5) راهنمای کیفیت کد (پیشنهادی)

- نام‌گذاری‌ها واضح و خوانا باشند.
- از کامنت‌های اضافه و غیرضروری پرهیز شود.
- اگر چیزی را تغییر می‌دهی، تست یا مستندات مرتبط اضافه کن.
- اگر ابزارهایی مثل `black` یا `ruff` در پروژه‌ات هست، همان‌ها را استفاده کن.



## 6) ساخت و ارسال Pull Request (PR)

1. تغییرات را اضافه کن:
```bash
git add .
```

2. commit بساز:
```bash
git commit -m "Add: <short description>"
```

3. push کن:
```bash
git push origin feature/<name>
```

4. در GitHub یک Pull Request باز کن و این موارد را ذکر کن:
- هدف تغییر (چرا؟)
- روش تست/تأیید (چطور؟)
- نکات خاص (اگر هست)



## 7) چه چیزهایی را می‌پذیریم؟

- باگ‌فیکس‌ها همراه با تست/نمونه
- بهبود مستندات (README و…)
- بهبود کارایی یا تمیزی کد با توضیح کوتاه
- ویژگی‌های جدید (در صورت هم‌خوانی با هدف پروژه)



## ارتباط
اگر ابهامی داری، یک Issue باز کن یا در PR توضیح بده.
